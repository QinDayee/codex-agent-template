# BRANCHING.md

## Branching Rules
- One feature or one phase per branch where possible
- Keep commits scoped and readable
- Review before merge
- Keep rollback simple

## Recommended Pattern
- feat/<short-name>
- fix/<short-name>
- chore/<short-name>

## Commit Pattern
- chore: initialize project template
- feat: implement phase 1 skeleton
- fix: correct null handling in member query
