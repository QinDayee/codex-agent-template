# TASKS.md

TASKS.md is the authoritative task board. Keep tasks concrete, testable, and resumable.

## Queue Rules
- Only one task should be `IN_PROGRESS` unless explicit parallel execution is approved.
- Every task must include owner, scope, done condition, and verification method.
- Move blocked work to a blocked task entry instead of hiding it in notes.
- Do not use TODO.md as the authoritative source of task status.
