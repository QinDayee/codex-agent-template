# RUNBOOK.md

## Purpose
Operational guide for how the current work should be executed.

## Standard Run Order
1. Read state and handoff files
2. Clarify requirement if needed
3. Produce or refresh plan
4. Split tasks
5. Implement one task or one phase
6. Review changes
7. Run quality gate
8. Update handoff and state
9. Continue

## When Blocked
- Record blocker in FAILURES.md
- Update HANDOFF.md
- Update STATE.md to BLOCKED
